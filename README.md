### Задание 1

Master-Slave (Один ведущий, один или несколько ведомых)

Master (ведущий) — единственный сервер, принимающий запросы на запись (INSERT, UPDATE, DELETE).
Slave (ведомый) — сервер, получающий копию данных с мастера только для чтения (SELECT).

Особенности:
- Однонаправленная синхронизация: изменения реплицируются только от мастера к слейву, но не наоборот.

Распределение нагрузки:
- Запись — только на мастер.
- Чтение — можно распределять по слейвам (например, для отчётов или аналитики).

Отказоустойчивость:
- При падении мастера требуется ручное или автоматическое (через менеджер репликации) переключение на слейв.
- Слейвы могут служить "горячим" резервом для восстановления.


Master-Master (Два или более ведущих серверов)
Каждый сервер является одновременно и мастером, и слейвом для других.
Запись возможна на любой узел, изменения синхронизируются между всеми узлами.

Особенности:
- Двунаправленная синхронизация: изменения с любого мастера реплицируются на все остальные узлы.

Распределение нагрузки:
- Запись и чтение возможны на любом узле.

Отказоустойчивость:
- При падении одного мастера остальные продолжают работать.
- Автоматическое переключение нагрузки.

Проблемы:
- Сложность настройки и поддержки.
- Риск несогласованности данных при высокой нагрузке.
